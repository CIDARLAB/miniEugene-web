Templating Constraints

Consider the following required design: 

p, r, c, t, p, r, c, t
miniEugene provides Templating Constraints for the specification of such patterns.

Specifically:

  • A Template constrains designs to comply with a specified list of components.
    Template constraints require that the Length of the design is a multiple of the number of components constraint by the template. Otherwise, no solution is found.

    Examples:
    • With a given length of 12 ( = 3 * 4), the template constraint 
      template p, r, c, t
      produces the following 4096 designs ('-' denotes reverse orientation):
       p,  r,  c,  t,  p,  r,  c,  t,  p,  r,  c,  t
 p,  r,  c,  t,  p,  r,  c,  t,  p,  r,  c, -t
 p,  r,  c,  t,  p,  r,  c,  t,  p,  r, -c,  t
 p,  r,  c,  t,  p,  r,  c,  t,  p,  r, -c, -t
...
-p, -r, -c, -t, -p, -r, -c, -t, -p, -r, -c,  t
-p, -r, -c, -t, -p, -r, -c, -t, -p, -r, -c, -t
    • A selection provides a list of possible components. The selection operator ("[ ]") selects one single component from the provided list. With a given length of 2, the template constraint 
      template [p1|p2], [c1|c2]
      produces the following 64 designs ('-' denotes reverse orientation):
       p1,  c1
 p1, -c1
-p1,  c1
-p1, -c1
 p2,  c1
 ...
 p2,  c2
-p2, -c2

  • A Sequence restricts that a set of components that must occur in a particular order. Sequences require that the Length of the design is greater or equal then the number of components constraint by a sequence. Otherwise, no solution is found.

    Examples:
    • The
      sequence p, r, c, t
      constraint results in the following 3,072 designs when given a length of 6 ('_' denotes place holders and '-' denotes reverse orientation):
       p,  r,  c, t, _, _
-p,  r,  c, t, _, _
 p, -r,  c, t, _, _
 p,  r, -c, t, _, _
...
_,  p,  r, c, t, _
_, -p,  r, c, t, _
_,  p, -r, c, t, _
...
_, _,  p,  r,  c,  t
_, _, -p,  r,  c,  t
...
_, _, -p, -r, -c, -t
    • The selection operator can also be utilized within sequence constraints. With a given length of 1, the 
      sequence [p|r|c|t]
      sequence constraint enumerates the following designs ('-' denotes reverse orientation):
       p
-p
 c
-c

    Note!
    • Templating Constraints can be utilized in conjunction (and disjunction) with all other provided constraints.
    • Templating Constraints are syntactic sugar of miniEugene and can degrade the efficiency of the design space exploration process.
    • Templating Constraints do not support the specification of desired orientations of the components. Additional Orientation Constraints must be specified to restrict or ensure desired orientations.

    See the Examples section for more detailed examples.
